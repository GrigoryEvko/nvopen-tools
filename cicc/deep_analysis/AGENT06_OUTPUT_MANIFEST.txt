ALGORITHM QUICK REFERENCE
================================================================================

Liveness Analysis:
  Input: Basic blocks with instructions
  Output: LIVE_IN and LIVE_OUT sets
  Complexity: O(N * E) iterations
  Result: Determines which variables are live at each point

Control Dependence:
  Method: PostDominatorTree computation
  Result: Determines which instructions depend on branches
  Application: ADCE marks control-dependent code as live

Worklist Algorithm:
  Type: Fixed-point iteration
  Operation: Breadth-first or depth-first traversal
  Convergence: Guaranteed (monotonic)
  Typical Iterations: 5-20 for realistic functions

MemorySSA:
  Representation: SSA form for memory operations
  Benefit: O(1) reachability checking
  Application: DSE uses for efficient store elimination

CFG Reachability:
  Method: Dominator tree or forward traversal
  Result: Identifies unreachable blocks
  Application: SimplifyCFG removes unreachable code
